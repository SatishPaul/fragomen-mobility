First-time administrator activation

1

Open only the newest email

Use the latest invitation or password-reset email sent to forweekend@gmail.com. Older links expire when a newer email is requested; check spam or junk if needed.

2

Verify the production domain and choose a password

The link must open https://fragomen-mobility.vercel.app. The app securely completes the link even when Supabase initially opens the home page. Set a password of at least eight characters.

Expected: the reset-password screen appears, then the app redirects to Dashboard after the password is saved.
3

Sign in

At /login, enter the invited email and new password.

Expected: Dashboard, Admin, Profile, and Sign out controls appear in the header.

Prepare demo media

  • Use 3 to 5 JPG or PNG photos with a clear sequence.
  • Optionally include one short MP4 clip encoded as H.264.
  • Use non-sensitive sample content; source media remains in the browser.
  • Allow popups only if you intentionally test social account connection.

Administrator path

Invite and manage users

02

Run this section while signed in as forweekend@gmail.com. The shield icon in the header opens Administration.

1

Open Administration

Select the shield icon or open /admin.

Expected: "Users and access" appears with Invite a user and Users sections.
2

Invite with the correct role

Enter the email and display name, then choose Regular user or Administrator. Keep administrator access limited to trusted operators.

Expected: "Invitation sent" appears. If the Auth identity already exists without a profile, the app recovers it and sends a password-setup email.
3

Set the monthly AI token limit

The 100,000 default is the monthly ceiling for remote AI input and output tokens. Choose a lower limit when appropriate; set 0 to block remote AI use.

4

Adjust access and publishing permission

Change role, quota, or Active status and select Save. For regular users, selected social accounts are destinations they may publish to only after explicit confirmation; assignment never posts automatically.

5

Delete a mistyped or obsolete user

Select Delete and review the permanent-removal warning. This removes the account, private videos, usage, and publishing records. You cannot delete, disable, or demote yourself, and one active administrator must remain.

Publishing authorization is deny-by-default Assignment grants permission to use a destination; it never creates a post by itself. Leave all accounts unassigned for a render-only demo.

Admin safety notes

  • Your signed-in administrator cannot be deleted, disabled, or demoted.
  • The system prevents removal of the last active administrator.
  • Never share the Supabase service-role key or Vercel environment values.
  • Use separate invited accounts for each tester; do not share passwords.
  • Deactivate an account to preserve its history; delete only when permanent removal is intended.

Core demo

Create the narrated video

03

Open /create. The six-step workspace advances from Upload through Publish. Complete the first five steps for a safe end-to-end demo.

1

Add your media

Upload 3 to 5 photos or short clips. Reorder them with Move earlier/Move later and remove any unwanted item.

Expected: asset count appears and Format & style unlocks.
2

Choose format and style

Select the target frame and one visual style. For a quick demo, use a short vertical or square output and keep the total under 30 seconds.

3

Generate narration

Select the script-generation action. The app analyzes downscaled frames and writes one editable narration line per scene. Review and edit every line.

Expected: one script scene per uploaded asset; token use is recorded.
4

Generate voiceover

Choose a voice, optionally preview it, then generate all voiceovers. The first browser-based voice run may download and cache the model.

Expected: each narrated scene receives an audio duration and Render unlocks.
5

Render video

Select Render video and keep the tab open. Rendering runs locally in the browser and may take several minutes.

Expected: progress reaches complete, a video preview appears, and the finished MP4 is saved privately.
6

Download and inspect

Select the Download control. Watch the full MP4 and verify order, crop, narration, timing, music level, watermark, and outro.

Core demo completion point Stop here for a no-side-effect demonstration. You have tested authentication, AI analysis, script generation, voice generation, browser rendering, private storage, and download without publishing anything.

Optional external action

Social publishing test

05
This section can create a real public post Do not continue unless the destination account owner has approved the exact video, caption, title, and timing. VideoMaker never publishes automatically; the tester must explicitly confirm the final action.

Prerequisites

  • Outstand is configured and the intended social account is connected.
  • An administrator assigned that exact account to the signed-in user.
  • The finished video was reviewed and approved.
  • The caption is final; YouTube and Vimeo also require a title.
  • The account owner approves a real immediate post. Scheduling is not available.

Publish only after approval

1

Load connected accounts

In Step 6, refresh accounts if needed. If connecting a new account, allow the OAuth popup and complete provider consent.

2

Select the exact destination

Verify both platform and account name. Do not select extra destinations.

3

Enter post content

Add the caption and, where required, the title. Keep combined content under 5,000 characters.

4

Review and publish

Select the publish action. Read the confirmation dialog, verify every named account, and confirm only when approved.

Expected: the MP4 uploads directly to Outstand, then per-account status appears.
5

Verify outcomes

Use Resume status if processing continues. Retry failed accounts only; successful accounts are excluded to avoid duplicates.

6

Review analytics

On Dashboard, open native post links and select Refresh performance for published items. Metrics depend on platform permissions.

Never test publishing casually Closing a confirmation dialog or leaving destination accounts unselected is the correct way to demonstrate publishing controls without creating an external post.
